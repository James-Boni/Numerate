Stage 2 is complete. Implement Stage 3 for Numerate.

Context:
Numerate already has a billing abstraction and provider pattern in place. Do not rewrite the architecture. Plug RevenueCat into the existing system.

Existing architecture that must be preserved:
- IBillingService contract already exists
- MockBillingService remains active for browser/dev
- RevenueCatCapacitorService currently exists as a stub
- SubscriptionProvider and useSubscription() already power the paywall and gated screens
- On login, billingService.logIn(uid) should be used
- On logout, billingService.logOut() should be used
- Single entitlement model: premium
- Supabase is NOT a source of truth for billing
- RevenueCat should only be used in native/Capacitor mode

Goal of this stage:
Replace the RevenueCatCapacitorService stub with a real implementation using the RevenueCat Capacitor SDK, while leaving browser/dev behavior untouched.

10. Keep all current mock/browser functionality intact.
